## Design Patterns Covered

1. **Adapter Pattern** - Allows incompatible interfaces to work together
2. **Builder Pattern** - Constructs complex objects step by step
3. **Command Pattern** - Encapsulates requests as objects
4. **Decorator Pattern** - Adds functionality to objects dynamically
5. **Dependency Injection** - Manages dependencies between components
6. **Factory Method Pattern** - Creates objects without specifying exact classes
7. **MVC Pattern** - Separates Model, View, and Controller concerns
8. **Observer Pattern** - Notifies multiple objects about state changes
9. **Proxy Pattern** - Controls access to another object
10. **Singleton Pattern** - Ensures only one instance of a class
11. **Strategy Pattern** - Encapsulates algorithms as interchangeable objects

## Getting Started

Clone the repository:
```bash
git clone https://github.com/sakshiagrawal890/Cognizant-java-fse.git
cd Cognizant-java-fse
```

Compile and run examples:
```bash
javac "WEEK-1/Engineering concepts/Design pattern and principles/AdapterPatternExample.java"
java -cp "WEEK-1/Engineering concepts/Design pattern and principles" AdapterPatternExample
```
